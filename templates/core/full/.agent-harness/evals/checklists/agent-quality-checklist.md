# Agent Quality Checklist

- Did the agent read the relevant repository context?
- Did it identify the current lifecycle stage when product work was involved?
- Did it read the relevant stage gate before moving stages?
- Did it update the relevant `.agent-harness/project/` memory when decisions changed?
- Did it create or update a spec for large product, architecture, or release changes?
- Did it keep active plans current for multi-session work?
- Did it treat external superpowers as optional and lower priority than harness policy?
- Did it keep changes scoped?
- Did it run verification?
- Did it report skipped checks or residual risks?
- Did it update handoff notes when needed?
