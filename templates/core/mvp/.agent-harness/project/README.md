# Project Memory

This directory is the project memory for the harness lifecycle.

Keep entries concise and update them as decisions change. The goal is not to
fill every file before coding. The goal is to make the current stage explicit,
capture the evidence behind decisions, and keep future agents aligned.

Start with:

1. `idea/idea-brief.md`
2. `discovery/problem-validation.md`
3. `product/mvp-scope.md`
4. `architecture/domain-model.md`
5. `execution/tasks.md`
6. `evaluation/metrics.md`
7. `operations/deployment.md`
8. `growth/growth-plan.md`

Then fill deeper files only when the project needs that detail.
