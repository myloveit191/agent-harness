# Feedback Loop

Use this workflow to turn evaluation results into better product decisions.

## Flow

```text
Collect feedback
  -> Identify patterns
  -> Compare against hypothesis
  -> Decide continue, adjust, narrow, or stop
  -> Update project memory
  -> Create tasks or revise roadmap
```

## Inputs

- User feedback.
- Product metrics.
- Technical metrics.
- Support issues or incidents.

## Outputs

- Updated assumptions.
- Updated MVP or roadmap.
- New tasks.
- Postmortem or learning notes when needed.
