# Phase To Task Workflow

Use this workflow to convert roadmap phases into executable work.

## Flow

```text
Phase
  -> Epic
  -> User Story
  -> Task
  -> Acceptance Criteria
  -> Verification
  -> Release Checklist
```

## Rules

- A phase has one outcome and clear exit criteria.
- An epic groups related user stories.
- A user story describes user intent and value.
- A task is small enough to complete and verify.
- Acceptance criteria are observable.
- Verification is named before implementation starts.

## Task Output

Each task should include:

- Goal.
- Scope.
- Acceptance criteria.
- Verification command or manual check.
- Status.
