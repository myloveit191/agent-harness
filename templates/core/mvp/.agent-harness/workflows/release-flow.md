# Release Flow

Use this workflow when preparing, shipping, or rolling back a release.

## Before Release

- Confirm scope against MVP, roadmap, or release goal.
- Run required verification.
- Review configuration and migrations.
- Confirm deployment and rollback steps.
- Confirm monitoring and incident ownership.

## Release

- Deploy using the documented deployment path.
- Smoke test critical user flows.
- Watch logs, metrics, and alerts.

## After Release

- Record release notes.
- Capture regressions and follow-up work.
- Update project memory when release outcomes change decisions.
