# Feature Workflow

Use this workflow for user-visible behavior or new capability.

1. Clarify the expected behavior and boundaries.
2. Inspect related code, tests, and documentation.
3. Plan the implementation and test coverage.
4. Add or update tests before or alongside the behavior change.
5. Implement in small, reviewable steps.
6. Run verification.
7. Document any setup, migration, or follow-up work.
